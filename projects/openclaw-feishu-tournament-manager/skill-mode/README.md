# Tournament Manager - Skill 模式

基于 OpenClaw + 飞书机器人 + 多维表格的炉石赛事管理系统。

## 功能特性

- ✅ 多赛事并发管理
- ✅ 管理员指令（/检查注册、/暂停顺延、/恢复顺延、/手动签到、/重发战报、/公示卡组）
- ✅ 队长签到（@机器人回复"签到"）
- ✅ 比分监听和自动顺延
- ✅ 战报发送
- ✅ 卡组公示

## 技术架构

```
用户 → 飞书群 → @吵吵 → OpenClaw → hs-esports agent → Bitable
```

## 核心流程

### 1. 手动签到流程

```
1. 管理员：/手动签到 2
2. 吵吵发送：【第2场签到】@队长A @队长B 请@吵吵 回复'签到'
3. 队长：@吵吵 签到
4. 系统验证身份，更新Bitable
5. 吵吵发送：✅ 测试战队A 已签到
```

### 2. 自动顺延流程

```
1. 第1场比分更新为 2:0（达到赛点）
2. 系统检查：顺延开关开启、存在下一场
3. 第1场标记为"已结束"、"已触发顺延"
4. 第2场状态设为"待签到"
5. 吵吵自动发送第2场签到通知
```

### 3. 战报发送流程

```
1. 管理员：/重发战报 2
2. 系统查询第2场比分和记录
3. 吵吵发送战报卡片到群聊
```

## 注意事项

1. **权限要求**：管理员需在"管理员白名单"表中配置
2. **顺延开关**：赛事配置表中的"顺延开关"控制自动顺延
3. **赛点阈值**：默认2分，可在对阵表中配置
4. **消息格式**：使用 Markdown 格式，支持 @ 提醒

## 限制说明

1. **交互卡片**：OpenClaw 不支持发送飞书交互卡片，使用纯文本替代
2. **消息监听**：队长需 @机器人 回复，系统才能实时接收
3. **回调机制**：不支持卡片按钮点击回调
