# 管理员指令参考

## 指令列表

| 指令 | 参数 | 权限 | 说明 |
|------|------|------|------|
| /检查注册 | 无 | 赛事管理员+ | 检查当前赛事的战队和选手注册情况 |
| /暂停顺延 | 无 | 赛事管理员+ | 暂停自动顺延功能 |
| /恢复顺延 | 无 | 赛事管理员+ | 恢复自动顺延功能 |
| /手动签到 [场次] | 场次序号 | 赛事管理员+ | 手动触发指定场次的签到 |
| /重发战报 [场次] | 场次序号 | 赛事管理员+ | 重新发送指定场次的战报 |
| /公示卡组 | 无 | 赛事管理员+ | 公示已审核通过的卡组 |

## 指令详细说明

### /检查注册

**功能**：检查当前赛事的战队和选手注册情况

**流程**：
1. 查询赛事配置表，确认赛事存在
2. 查询战队主档案库，统计战队数量
3. 查询选手主档案库，统计选手数量
4. 返回统计结果

**返回示例**：
```
注册检查完成，当前共有 8 支战队，24 名选手。
```

### /暂停顺延

**功能**：暂停自动顺延功能

**流程**：
1. 验证管理员权限
2. 查询当前赛事配置
3. 更新"顺延开关"为 false
4. 记录审计日志
5. 返回确认消息

**返回示例**：
```
已暂停本赛事自动顺延，后续比分变化将不再自动触发下一场签到。
```

### /恢复顺延

**功能**：恢复自动顺延功能

**流程**：
1. 验证管理员权限
2. 查询当前赛事配置
3. 更新"顺延开关"为 true
4. 记录审计日志
5. 返回确认消息

**返回示例**：
```
已恢复本赛事自动顺延。历史漏发场次如需补发，请使用 /手动签到。
```

### /手动签到 [场次]

**功能**：手动触发指定场次的签到

**参数**：
- 场次序号：数字，如 2 表示第2场

**流程**：
1. 验证管理员权限
2. 查询指定场次信息
3. 检查场次状态是否为"未开始"
4. 更新签到状态为"已下发"
5. 发送签到卡片到群聊（@双方队长）
6. 记录审计日志

**返回示例**：
```
已手动下发第 2 场签到卡片。
```

**错误情况**：
- 场次不存在："未找到第 X 场对阵，请检查场次序号"
- 场次已开始："该场次已开始，无法手动签到"

### /重发战报 [场次]

**功能**：重新发送指定场次的战报

**参数**：
- 场次序号：数字

**流程**：
1. 验证管理员权限
2. 查询指定场次信息
3. 检查场次状态是否为"已结束"
4. 重新发送战报卡片到群聊
5. 记录审计日志

**返回示例**：
```
已重新发送第 4 场战报。
```

**错误情况**：
- 场次不存在："未找到对应场次，请检查场次序号或 MatchID"
- 场次未结束："该场次尚未结束，无法重发战报"

### /公示卡组

**功能**：公示已审核通过的卡组

**流程**：
1. 验证管理员权限
2. 查询赛事配置，检查"公示开关"是否为 true
3. 查询所有已审核通过的卡组
4. 生成卡组公示卡片
5. 发送到群聊
6. 记录审计日志

**返回示例**：
```
已公示卡组，共 8 套卡组。
```

**错误情况**：
- 公示开关未开启："当前未开启卡组公示，请先确认赛事配置中的公示开关"

## 权限级别

| 级别 | 说明 |
|------|------|
| 超级管理员 | 可管理所有赛事，可修改管理员白名单 |
| 赛事管理员 | 可管理指定赛事，可执行所有指令 |
| 只读 | 仅可查看信息，不可执行修改操作 |

## 审计日志

所有指令执行都会记录到审计日志表，包括：
- 操作类型
- 操作人OpenID
- 目标赛事ID
- 目标场次（如适用）
- 操作时间
- 操作详情（JSON格式）
- 操作结果（成功/失败）
- 失败原因（如失败）
