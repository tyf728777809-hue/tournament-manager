# 个人赛联调清单 v1

> 日期：2026-03-28

## 当前目标

验证个人赛最小闭环：

1. 报名提交
2. 管理员审核通过
3. 收到个人赛 4 套卡组提交入口
4. 提交 4 套卡组
5. 管理员审核通过

---

## 一、前置数据

### 1. 赛事表 `tournaments`
需存在一条个人赛测试赛事，例如：
- `tournament_uid = HS202603-TEST-SOLO`
- `tournament_name = 三月月赛测试-个人赛`
- `tournament_type = solo`
- `public_deck_publish_mode = manual_publish`（可选，便于后面测试公示确认）
- 报名与卡组截止时间未过期

### 2. 管理员表 `tournament_admins`
需存在：
- `tournament_uid = HS202603-TEST-SOLO`
- `user_open_id = ou_914e6141a81eb6da2602875aee631269`
- `role = owner`
- `status = active`

---

## 二、联调步骤

### Step 1：提交个人赛报名
使用 action：`submit_player_registration_form`

预期结果：
- `players` 有该选手资料
- `registrations` 新建一条记录
- `registration_status = submitted`
- 返回 `registration_uid`

### Step 2：管理员审核通过报名
使用 action：`approve_registration`

预期结果：
- `registration_status = approved`
- `review_status = approved`
- 选手收到“报名已通过审核”通知
- 选手收到“提交4套卡组”入口卡片

### Step 3：提交个人赛 4 套卡组
使用 action：`submit_player_deck_form`

预期结果：
- `deck_submissions` 新建 1 条记录
- `decks` 新建 4 条记录
- `submission_status = submitted`
- 若职业重复或少于 4 套，应明确报错

### Step 4：管理员审核通过卡组
使用 action：`approve_deck_submission`

预期结果：
- `submission_status = approved`
- `is_current_effective_version = true`
- 选手收到“卡组已通过审核”通知
- 若赛事配置 `manual_publish`，管理员收到公示确认卡片

---

## 三、当前风险点

1. 真实线上字段名可能与当前实现假设略有出入
   - 比如 `registration_deadline`
   - `deck_deadline`
   - `player_uid / entity_uid`

2. 真实表单回调字段可能不是当前假设命名
   - 报名表：`real_name / player_id / nickname / bnet_id`
   - 卡组表：当前已兼容数组和部分平铺字段

3. `registration_uid` / `player_uid` 的自动生成规则，目前是工程内约定，不一定和你最终想要的命名完全一致

---
