# 个人赛小局明细落地说明 v1

> 日期：2026-03-28

## 1. 本轮目标

把个人赛 BO5 征服在 `matches` 级写回之外，再往前推进一层：

- 为每一小局保留独立明细（`match_games`）
- 为每局后的职业池状态保留快照（`match_class_states`）
- 采用 **有表就写、没表就降级不阻塞** 的方式接入当前代码

---

## 2. 当前代码已完成

已在 `callback-handler.js` 中实现：

### 2.1 可配置表 ID
通过环境变量读取：

- `HS_MATCH_GAMES_TABLE_ID`
- `HS_MATCH_CLASS_STATES_TABLE_ID`

若未配置：
- 不阻塞 `record_solo_game_result`
- 仍会正常写回 `matches`
- 只是跳过小局明细和职业池快照表写入

### 2.2 幂等 Upsert
新增 `safeUpsertByUid()`：

- `match_games` 按 `match_game_uid` 幂等写入
- `match_class_states` 按 `match_class_state_uid` 幂等写入

这样即使同一局重复回放写入，也不会无限创建重复记录。

### 2.3 当前写入字段

#### `match_games` 建议字段
- `match_game_uid`
- `match_uid`
- `tournament_uid`
- `game_number`
- `side_a_class`
- `side_b_class`
- `winner_side`
- `loser_side`
- `winner_class`
- `loser_class`
- `result_status`
- `ended_at`
- `internal_note`

#### `match_class_states` 建议字段
- `match_class_state_uid`
- `match_uid`
- `tournament_uid`
- `game_number`
- `side_a_locked_class`
- `side_b_locked_class`
- `side_a_available_classes`
- `side_b_available_classes`
- `side_a_conquered_classes`
- `side_b_conquered_classes`
- `side_a_wins`
- `side_b_wins`
- `winner_side`
- `locked_at`

---

## 3. 当前行为

每次执行 `record_solo_game_result`：

1. 先更新 `matches` 中的整场状态
2. 再尝试写：
   - `match_games` 一条小局明细
   - `match_class_states` 一条该局后的职业池快照
3. 若上述两张表未建 / 未配置，不影响主流程继续
