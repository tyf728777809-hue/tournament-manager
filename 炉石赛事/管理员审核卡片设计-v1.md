# 管理员审核卡片设计 v1

> 炉石赛事自动化系统 - 管理员审核界面设计
> 版本: v1.0 | 日期: 2026-03-28

---

## 1. 设计原则

### 1.1 审核场景
- **批量处理**: 管理员需要快速处理多个待审核项
- **信息完整**: 一眼看到所有关键信息，无需跳转
- **操作便捷**: 一键通过/驳回，支持快捷回复
- **可追溯**: 审核记录完整保存

### 1.2 卡片类型
| 类型 | 用途 | 触发条件 |
|------|------|----------|
| 报名审核卡片 | 审核选手/战队报名 | registrations.status = pending_review |
| 卡组审核卡片 | 审核卡组提交 | deck_submissions.status = pending_review |
| 争议裁决卡片 | 处理争议申诉 | disputes.status = admin_review |

#### 字段映射

| 卡片字段 | 数据来源表 | 字段名 |
|----------|-----------|--------|
| 真实姓名 | players | real_name |
| 选手ID | players | player_id |
| 战网ID | players | battle_net_id |
| 参赛赛事 | tournaments | name (通过 tournament_id 关联) |
| 报名时间 | registrations | created_at |
| 参赛宣言 | registrations | declaration |
| 资料来源 | registrations | source_type, source_id |

#### 审核动作

| 动作 | API调用 | 状态变更 |
|------|---------|----------|
| 通过 | PATCH registrations | pending_review → approved |
| 驳回 | PATCH registrations + 发送通知 | pending_review → rejected |
| 需要补充 | 发送私聊消息 | pending_review (保持) |

## 6. 卡片触发机制

### 6.1 主动推送
当有新待审核项时，机器人主动推送卡片到管理员私聊：

```javascript
// 伪代码
if (newRegistration.status === 'pending_review') {
  // 查询管理员open_id
  const admins = await getTournamentAdmins(tournament_id);
  
  // 构建审核卡片
  const card = buildRegistrationReviewCard(registrationData);
  
  // 推送到每个管理员
  for (const admin of admins) {
    await sendCardMessage({
      receive_id: admin.open_id,
      receive_id_type: 'open_id',
      content: card
    });
  }
}
```

### 6.2 被动查询
管理员输入"查看待办"时返回工作台卡片：

```javascript
// 查询所有pending状态的记录
const pendingRegistrations = await queryRecords({
  table: 'registrations',
  filter: { status: 'pending_review' }
});

const pendingDecks = await queryRecords({
  table: 'deck_submissions', 
  filter: { status: 'pending_review' }
});

const pendingDisputes = await queryRecords({
  table: 'disputes',
  filter: { status: 'admin_review' }
});

// 构建工作台卡片
const workbenchCard = buildWorkbenchCard({
  registrations: pendingRegistrations.length,
  decks: pendingDecks.length,
  disputes: pendingDisputes.length
});
```

## 7. 按钮回调处理

### 7.1 回调数据结构

```json
{
  "action": "approve_registration",
  "registration_id": "reg_001",
  "player_uid": "player_001",
  "admin_open_id": "ou_xxx",
  "timestamp": "2026-03-28T12:00:00+08:00"
}
```

### 7.2 处理流程

```javascript
// 处理审核通过
async function handleApproveRegistration(data) {
  // 1. 更新registrations表
  await updateRecord({
    table: 'registrations',
    record_id: data.registration_id,
    fields: {
      status: 'approved',
      reviewed_at: new Date().toISOString(),
      reviewed_by: data.admin_open_id
    }
  });
  
  // 2. 发送通知给选手
  await sendNotification({
    receive_id: data.player_uid,
    message: '✅ 您的报名已通过审核！'
  });
  
  // 3. 更新卡片状态（显示"已处理"）
  await updateCardMessage({
    message_id: data.message_id,
    content: buildApprovedCard(data)
  });
}
```
