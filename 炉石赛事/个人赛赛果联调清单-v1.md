## 3. 典型命令

### 3.1 提交赛果

```bash
node scripts/hs-solo-result-smoke.js submit \
  --match MATCH-HS202603-TEST-SOLO-001 \
  --operator <side_a_or_side_b_open_id> \
  --result "小甲 3:1 小乙"
```

预期：
- `result_reports` 新建 / 更新 1 条记录
- `report_status = awaiting_opponent_confirmation`
- 对手收到确认卡片

### 3.2 对手确认

```bash
node scripts/hs-solo-result-smoke.js confirm \
  --match MATCH-HS202603-TEST-SOLO-001 \
  --report <result_report_uid> \
  --operator <opponent_open_id>
```

若测试样本双方共用同一个 open_id，可临时使用受控联调模式：

```bash
HS_SOLO_RESULT_TEST_BYPASS=1 \
node scripts/hs-solo-result-smoke.js confirm \
  --match MATCH-HS202603-TEST-SOLO-001 \
  --report <result_report_uid> \
  --operator <same_open_id> \
  --as-side side_b
```

预期：
- `result_reports.report_status = opponent_confirmed`
- `matches.match_status = completed`
- `matches.result_status = confirmed`

### 3.3 对手拒绝

```bash
node scripts/hs-solo-result-smoke.js reject \
  --match MATCH-HS202603-TEST-SOLO-001 \
  --report <result_report_uid> \
  --operator <opponent_open_id> \
  --reason "比分不一致，申请管理员核实"
```

若测试样本双方共用同一个 open_id，可临时使用：

```bash
HS_SOLO_RESULT_TEST_BYPASS=1 \
node scripts/hs-solo-result-smoke.js reject \
  --match MATCH-HS202603-TEST-SOLO-001 \
  --report <result_report_uid> \
  --operator <same_open_id> \
  --as-side side_b \
  --reason "比分不一致，申请管理员核实"
```

预期：
- `result_reports.report_status = opponent_rejected`
- `matches.match_status = disputed`
- `disputes` 自动创建关联记录
- 管理员收到升级卡片

### 3.4 超时升级管理员

```bash
node scripts/hs-solo-result-smoke.js timeout \
  --match MATCH-HS202603-TEST-SOLO-001 \
  --report <result_report_uid> \
  --operator <admin_open_id> \
  --force true \
  --reason "联调时强制模拟超时"
```

预期：
- `result_reports.report_status = awaiting_admin_review`
- `opponent_confirmation_status = timeout`
- `disputes` 自动创建关联记录
- 管理员收到升级卡片

### 3.5 管理员确认

```bash
node scripts/hs-solo-result-smoke.js admin-confirm \
  --match MATCH-HS202603-TEST-SOLO-001 \
  --report <result_report_uid> \
  --admin <admin_open_id>
```

预期：
- `result_reports.report_status = admin_confirmed`
- `matches.match_status = completed`
- `matches.result_status = confirmed`
