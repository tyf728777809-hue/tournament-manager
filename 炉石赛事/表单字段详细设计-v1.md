# 表单字段详细设计 v1

> 炉石赛事自动化系统 - P0 表单字段与校验设计
> 版本: v1.0 | 日期: 2026-03-28

---

## 1. 设计目标

本文件细化 P0 阶段 5 类核心表单：
1. 个人赛报名表单
2. 战队赛报名表单
3. 个人赛卡组提交表单
4. 战队赛卡组提交表单
5. 争议提交表单

每个表单包含：
- 字段清单
- 字段类型
- 是否必填
- 校验规则
- 对应数据落表
- 提交成功回执

---

## 2. 个人赛报名表单

### 2.1 表单定位
- 入口：个人赛报名入口卡片
- 目标表：`players` + `registrations`
- 适用对象：单人参赛选手

### 2.2 字段清单

| 分组 | 字段名 | 字段类型 | 必填 | 校验规则 | 落表 |
|------|--------|----------|------|----------|------|
| 基本信息 | 真实姓名 | 单行文本 | 是 | 2-20字 | players.real_name |
| 基本信息 | 选手ID | 单行文本 | 是 | 1-30字符，全局唯一优先 | players.player_id |
| 基本信息 | 昵称/展示名 | 单行文本 | 是 | 1-30字符 | players.nickname |
| 基本信息 | 战网ID | 单行文本 | 是 | 3-50字符，建议包含#编号 | players.bnet_id |
| 基本信息 | 飞书OpenID | 系统自动带入 | 否 | 不让用户手填 | players.feishu_open_id |
| 展示资料 | 荣誉 | 多行文本 | 否 | 最长500字 | players.honors |
| 展示资料 | 定妆照 | 附件/图片 | 否 | 仅图片，≤10MB | players.portrait_image |
| 赛事信息 | 参赛宣言 | 多行文本 | 否 | 最长200字 | registrations.snapshot_slogan |
| 赛事信息 | 赛事UID | 隐藏字段 | 是 | 由入口预置 | registrations.tournament_uid |
| 赛事信息 | entity_type | 隐藏字段 | 是 | 固定为player | registrations.entity_type |

### 2.3 前端交互建议
- 若存在历史资料：默认预填真实姓名、选手ID、昵称、战网ID、荣誉、定妆照
- 参赛宣言默认留空，作为赛事级快照
- 提交前展示“你本次提交将覆盖基础资料，并生成本次赛事报名快照”

### 2.4 提交校验

| 校验项 | 规则 | 失败提示 |
|--------|------|----------|
| 真实姓名 | 非空 | 请填写真实姓名 |
| 选手ID | 非空 | 请填写选手ID |
| 战网ID | 非空 | 请填写战网ID |
| 头像格式 | 仅图片 | 定妆照请上传图片文件 |
| 赛事有效性 | tournament_uid存在且状态允许报名 | 当前赛事暂不可报名 |

### 2.5 成功回执

```text
已收到你的个人赛报名信息。
当前状态：已提交，等待审核。
下一步：审核通过后，你将收到卡组提交入口。
如需补充资料，我会继续私聊你。
```

## 3. 战队赛报名表单

### 3.1 表单定位
- 入口：战队赛报名入口卡片
- 目标表：`teams` + `players` + `team_members` + `registrations` + `registration_members`
- 适用对象：战队队长

### 3.2 字段清单

#### A. 战队信息

| 字段名 | 类型 | 必填 | 校验规则 | 落表 |
|--------|------|------|----------|------|
| 战队名称 | 单行文本 | 是 | 2-40字符，建议全局唯一 | teams.team_name |
| 战队Logo | 附件/图片 | 否 | 仅图片，≤10MB | teams.team_logo |
| 战队宣言 | 多行文本 | 否 | 最长200字 | teams.team_slogan |
| 队长OpenID | 系统自动带入 | 是 | 当前提交人 | teams.captain_open_id |
| 赛事UID | 隐藏字段 | 是 | 由入口预置 | registrations.tournament_uid |

#### B. 队长资料

| 字段名 | 类型 | 必填 | 校验规则 | 落表 |
|--------|------|------|----------|------|
| 队长真实姓名 | 单行文本 | 是 | 2-20字 | players.real_name |
| 队长选手ID | 单行文本 | 是 | 1-30字符 | players.player_id |
| 队长昵称 | 单行文本 | 是 | 1-30字符 | players.nickname |
| 队长战网ID | 单行文本 | 是 | 3-50字符 | players.bnet_id |
| 队长荣誉 | 多行文本 | 否 | ≤500字 | players.honors |
| 队长定妆照 | 图片 | 否 | ≤10MB | players.portrait_image |

#### C. 首发成员（3人）
每人一组相同字段：
- 真实姓名（必填）
- 选手ID（必填）
- 昵称（必填）
- 战网ID（必填）
- 荣誉（可选）
- 定妆照（可选）
- 角色（隐藏：starter）

#### D. 替补成员（最多2人）
每人一组相同字段：
- 真实姓名（至少0，最多2组）
- 选手ID
- 昵称
- 战网ID
- 荣誉
- 定妆照
- 角色（隐藏：substitute）

### 3.3 提交校验

| 校验项 | 规则 | 失败提示 |
|--------|------|----------|
| 战队名称 | 非空 | 请填写战队名称 |
| 首发人数 | 必须恰好3人 | 首发人数必须为3人 |
| 替补人数 | 最多2人 | 替补人数最多2人 |
| 队内重复 | 选手ID/战网ID不能重复 | 队伍中存在重复成员信息 |
| 队长一致性 | 提交人即队长或具队长权限 | 当前仅支持队长提交 |

### 3.4 成功回执

```text
已收到你的战队报名信息。
当前状态：已提交，等待审核。
下一步：审核通过后，队长将收到战队卡组提交通知。
如需补充资料，我会继续私聊你。
```

## 4. 个人赛卡组提交表单

### 4.1 表单定位
- 入口：个人赛卡组提交入口卡片
- 目标表：`deck_submissions` + `decks`
- 适用对象：个人赛已审核通过的选手

### 4.2 表单字段

#### 公共字段

| 字段名 | 类型 | 必填 | 校验规则 | 落表 |
|--------|------|------|----------|------|
| registration_uid | 隐藏字段 | 是 | 必须对应已通过报名记录 | deck_submissions.registration_uid |
| tournament_uid | 隐藏字段 | 是 | 由入口预置 | deck_submissions.tournament_uid |
| entity_type | 隐藏字段 | 是 | 固定为player | deck_submissions.entity_type |
| submission_reason | 隐藏字段 | 是 | 默认initial_submission | deck_submissions.submission_reason |

#### 4套卡组字段（重复4组）

| 字段名 | 类型 | 必填 | 校验规则 | 落表 |
|--------|------|------|----------|------|
| 职业 | 单选 | 是 | 必须从赛事允许职业中选择 | decks.class_name |
| 卡组名称 | 单行文本 | 否 | 1-50字符 | decks.deck_name |
| 卡组代码 | 多行文本 | 是 | 非空，格式校验 | decks.deck_code |
| 卡组截图 | 附件/图片 | 否 | 仅图片，≤10MB | decks.deck_image |
| deck_order | 隐藏字段 | 是 | 1-4 | decks.deck_order |

### 4.3 提交校验

| 校验项 | 规则 | 失败提示 |
|--------|------|----------|
| 数量 | 必须4套 | 请提交4套卡组 |
| 职业去重 | 4套职业必须不同 | 4套卡组职业不能重复 |
| 卡组代码 | 非空 | 请补充卡组代码 |
| 截止时间 | 未超过deck_deadline | 卡组提交已截止 |
| 报名状态 | registration已approved | 当前无卡组提交资格 |

### 4.4 成功回执

```text
已收到你的4套卡组。
当前状态：已提交，正在检查与审核。
下一步：审核通过后进入保密存档；如有问题会私聊通知你修改。
```

## 5. 战队赛卡组提交表单

### 5.1 表单定位
- 入口：战队赛卡组提交入口卡片
- 目标表：`deck_submissions` + `decks`
- 适用对象：战队赛已审核通过的队长

### 5.2 表单字段

#### 公共字段
同个人赛，但 `entity_type = team`

#### 11/12套卡组字段（重复11或12组）

| 字段名 | 类型 | 必填 | 校验规则 | 落表 |
|--------|------|------|----------|------|
| 职业 | 单选 | 是 | 赛事允许职业范围 | decks.class_name |
| 卡组名称 | 单行文本 | 否 | 1-50字符 | decks.deck_name |
| 卡组代码 | 多行文本 | 是 | 非空 | decks.deck_code |
| 卡组截图 | 图片 | 否 | ≤10MB | decks.deck_image |
| deck_order | 隐藏字段 | 是 | 1-12 | decks.deck_order |

### 5.3 提交校验

| 校验项 | 规则 | 失败提示 |
|--------|------|----------|
| 数量 | 必须11或12套 | 请按赛事规则提交11/12套卡组 |
| 职业覆盖 | 满足赛事职业覆盖规则 | 当前卡组职业覆盖不完整 |
| 截止时间 | 未超过截止 | 卡组提交已截止 |
| 提交权限 | 仅队长可提交 | 当前仅支持队长统一提交 |

### 5.4 成功回执

```text
已收到战队卡组提交。
当前状态：已提交，正在检查与审核。
下一步：审核通过后将按赛事规则保密保存，待统一公示或进入BP流程。
```

## 6. 争议提交表单

### 6.1 表单定位
- 入口：私聊机器人输入“提交争议”
- 目标表：`disputes`
- 适用对象：参赛选手 / 队长

### 6.2 字段清单

| 字段名 | 类型 | 必填 | 校验规则 | 落表 |
|--------|------|------|----------|------|
| tournament_uid | 隐藏字段 | 是 | 系统预置 | disputes.tournament_uid |
| match_uid | 下拉选择/自动联想 | 否 | 优先从近期对局中选 | disputes.match_uid |
| 对局对象 | 单行文本 | 是 | 1-50字符 | disputes.opponent_name_snapshot |
| 争议类型 | 单选 | 是 | 从预置类型选 | disputes.dispute_type |
| 发生时间 | 日期时间 | 是 | 不晚于当前时间 | disputes.occurred_at |
| 详细说明 | 多行文本 | 是 | 20-2000字 | disputes.description |
| 诉求 | 多行文本 | 否 | ≤500字 | disputes.requested_resolution |
| 证据附件 | 附件 | 否 | 图片/视频/文件，≤20MB/个 | disputes.evidence_attachments |

### 6.3 争议类型建议选项
- 对局结果异议
- 掉线/断线争议
- BUG/异常情况
- 迟到/超时争议
- 违规行为举报
- 其他

### 6.4 提交校验

| 校验项 | 规则 | 失败提示 |
|--------|------|----------|
| 争议类型 | 必选 | 请选择争议类型 |
| 发生时间 | 必填 | 请填写争议发生时间 |
| 详细说明 | 不少于20字 | 请更详细说明争议情况 |
| 资格校验 | 必须为当前赛事相关人员 | 当前仅支持赛事相关人员提交争议 |

### 6.5 成功回执

```text
争议已提交。
当前状态：待审查。
下一步：系统会先结合规则与上下文生成初审建议，再交管理员处理。
如需补充证据，我会继续私聊你。
```

## 7. 字段与数据表映射策略

### 7.1 基础资料 vs 赛事快照
- `players` / `teams`：存长期复用资料
- `registrations` / `registration_members`：存本赛事快照
- `deck_submissions` / `decks`：存本赛事卡组版本
- `disputes`：存赛事争议闭环数据

### 7.2 推荐隐藏字段
所有表单建议由系统自动注入以下隐藏字段：
- tournament_uid
- registration_uid
- entity_type
- submitted_by_open_id
- source_channel
- source_message_id（如可取）
