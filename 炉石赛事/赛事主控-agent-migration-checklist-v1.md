# 赛事主控 Agent Migration Checklist v1

> 用途：指导“新赛事主控 agent + 新飞书 bot”从准备、灰度到正式接管的执行清单。

---

## 1. 迁移目标

- [ ] 个人助理与赛事主控职责分离
- [ ] 赛事系统由独立赛事主控 agent 接管
- [ ] 使用独立飞书机器人承接赛事入口
- [ ] 结构化数据仍以飞书多维表格 + 飞书文档为真相源
- [ ] 保留灰度与回滚路径，避免一次性硬切

---

## 2. 实例准备

- [ ] 已准备独立 OpenClaw 实例（推荐独立服务器/独立配置）
- [ ] 已准备独立 workspace
- [ ] 已准备独立 config
- [ ] 已准备独立 log / session / memory 边界
- [ ] 已确认不继承个人助理实例的私人长期记忆

---

## 3. 新 bot 准备

- [ ] 已创建新的飞书应用 / bot
- [ ] 已获取 appId / appSecret / verificationToken / encryptKey（如有）
- [ ] 已配置消息与回调权限
- [ ] 已明确新 bot 只服务赛事场景
- [ ] 已明确新 bot 不承接个人助理入口

---

## 4. 数据接入确认

- [ ] 已确认新赛事主控可访问所需 Bitable
- [ ] 已确认新赛事主控可访问所需飞书文档
- [ ] 已确认核心表结构与字段命名无偏差
- [ ] 已确认管理员身份映射方式
- [ ] 已确认测试赛事与正式赛事的隔离方式

---

## 5. 文档与知识交接

- [ ] 已交付 core file
- [ ] 已交付 operating rules
- [ ] 已交付 handover checklist
- [ ] 已交付 bootstrap context
- [ ] 已交付 SOP index
- [ ] 已确认新 agent 能按索引找到关键文档

---

## 6. 技术接线确认

- [ ] 已部署回调服务
- [ ] 已配置飞书事件订阅 URL
- [ ] 已通过 url_verification
- [ ] 已完成至少一轮 dry-run 验证
- [ ] 已完成至少一轮真实卡片回调验证
- [ ] 已确认日志可查看、异常可定位

---

## 7. 灰度阶段

### 建议灰度范围
- [ ] 仅一个测试赛事
- [ ] 仅一个测试群
- [ ] 仅一个管理员入口
- [ ] 仅最小闭环流程（报名/卡组/赛果/争议中的部分链路）

### 灰度观察点
- [ ] 是否出现串群/串赛事
- [ ] 是否出现错写表
- [ ] 是否依赖私人助理上下文才能推进
- [ ] 是否能独立输出待办与摘要
- [ ] 是否能正确升级人工确认

---

## 8. 正式切流前检查

- [ ] 新 bot 稳定可用
- [ ] 新 agent 可独立运行
- [ ] 核心链路已验证
- [ ] 管理员已知如何联系新 bot / 新主控
- [ ] 旧链路仍保留兜底能力
- [ ] 已制定回滚触发条件

---

## 9. 回滚预案

- [ ] 若新主控异常，如何暂停新入口
- [ ] 若回调异常，如何临时转人工
- [ ] 若写表异常，如何停止进一步扩散
- [ ] 若需要恢复旧入口，谁来执行、怎么执行
- [ ] 是否有观察期（建议 1~2 周）

---

## 10. 正式接管完成判定

只有当以下条件满足，才算迁移完成：
- [ ] 新赛事由新主控承接
- [ ] 个人助理不再承担赛事主控职责
- [ ] 新 bot 成为赛事默认入口
- [ ] 数据一致性稳定
- [ ] 没有出现跨赛事/跨上下文串线
- [ ] 出现异常时有稳定回滚路径

---

## 11. 一句话执行原则

> 先分边界，再建新入口；先灰度，再切流；先保回滚，再谈彻底替换。
