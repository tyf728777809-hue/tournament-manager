## 5. 当前遗留问题

1. 本地 `callback-handler.js` 直接运行依赖 `FEISHU_ACCESS_TOKEN`，当前 shell 环境未注入，无法直接用本地 Node 执行联调
2. 本地代码中的部分字段名假设与真实线上字段仍有差异（本轮已通过真实表字段校正了一部分）
3. Control UI 仍存在“页面能点但消息不刷新”的问题，会影响联调体验，但不影响飞书真实写表
4. 赛果主闭环已完成，后续遗留更适合转入线上接入、异常分支补齐或前端/Control UI 稳定性治理，而不是继续停留在本轮联调本身
