# P0 入口状态变化表 v1

> 炉石赛事自动化系统 - 第一阶段核心入口状态机设计
> 版本: v1.0 | 日期: 2026-03-28

---

## 1. 个人赛报名入口

### 1.1 状态定义

| 状态码 | 状态名 | 说明 |
|--------|--------|------|
| `not_started` | 未开始 | 用户未触发报名流程 |
| `profile_check` | 资料检查中 | 系统检查是否已有历史资料 |
| `reuse_select` | 复用选择 | 展示复用/修改/新建选项 |
| `form_filling` | 填写中 | 用户正在填写报名表单 |
| `submitted` | 已提交 | 表单已提交，待审核 |
| `pending_review` | 审核中 | 管理员审核中 |
| `approved` | 已通过 | 报名成功，获得参赛资格 |
| `rejected` | 已驳回 | 审核未通过，需修改重提 |
| `cancelled` | 已取消 | 用户主动取消或超时放弃 |

### 1.2 状态流转图

```
not_started
    ↓ (用户点击"个人赛报名")
profile_check ──→ 无历史资料 ──→ form_filling
    ↓ 有历史资料
reuse_select
    ├── 选择"直接沿用" ──→ submitted
    ├── 选择"修改资料" ──→ form_filling
    └── 选择"仅更新宣言" ──→ 简化表单 ──→ submitted

form_filling
    ├── 提交 ──→ submitted
    ├── 保存草稿 ──→ draft_saved
    └── 取消 ──→ cancelled

submitted ──→ pending_review
    ↓
    ├── 审核通过 ──→ approved
    └── 审核驳回 ──→ rejected ──→ form_filling (修改)
```

### 1.3 触发条件与动作

| 当前状态 | 触发条件 | 动作 | 下一状态 |
|----------|----------|------|----------|
| `not_started` | 用户点击报名卡片/输入"报名" | 查询players表 | `profile_check` |
| `profile_check` | 查询结果：无记录 | 推送空白表单 | `form_filling` |
| `profile_check` | 查询结果：有记录 | 推送复用选择卡片 | `reuse_select` |
| `reuse_select` | 用户选择"直接沿用" | 复制历史资料到registrations | `submitted` |
| `reuse_select` | 用户选择"修改资料" | 推送预填表单 | `form_filling` |
| `reuse_select` | 用户选择"仅更新宣言" | 推送简化表单 | `form_filling` |
| `form_filling` | 用户提交表单 | 写入players + registrations | `submitted` |
| `form_filling` | 用户保存草稿 | 暂存数据 | `draft_saved` |
| `submitted` | 系统自动流转 | 通知管理员审核 | `pending_review` |
| `pending_review` | 管理员通过 | 更新状态，通知选手 | `approved` |
| `pending_review` | 管理员驳回 | 更新状态，说明原因 | `rejected` |
| `rejected` | 用户修改重提 | 更新数据 | `submitted` |

### 1.4 数据写入

| 状态 | 写入表 | 写入内容 |
|------|--------|----------|
| `form_filling` → `submitted` | players | 真实姓名、选手ID、昵称、战网ID、荣誉、定妆照 |
| `form_filling` → `submitted` | registrations | 赛事ID、player_uid、参赛宣言、报名状态 |
| `reuse_select` → `submitted` | registrations | 快照复制历史资料，标记复用来源 |
| `approved` | registrations | 更新状态为approved，记录审核时间 |

## 2. 战队赛报名入口

### 2.1 状态定义

| 状态码 | 状态名 | 说明 |
|--------|--------|------|
| `not_started` | 未开始 | 未触发报名流程 |
| `team_profile_check` | 战队资料检查 | 检查战队历史资料 |
| `team_reuse_select` | 战队复用选择 | 沿用/修改/新建战队 |
| `team_form_filling` | 战队信息填写 | 填写战队基础信息 |
| `member_intake` | 成员录入 | 录入首发+替补成员 |
| `member_profile_check` | 成员资料检查 | 检查每个成员历史资料 |
| `member_reuse_select` | 成员复用选择 | 每个成员的复用选择 |
| `member_form_filling` | 成员信息填写 | 填写成员详细信息 |
| `roster_confirm` | 名单确认 | 确认最终出战名单 |
| `submitted` | 已提交 | 战队报名已提交 |
| `pending_review` | 审核中 | 等待管理员审核 |
| `approved` | 已通过 | 报名成功 |
| `rejected` | 已驳回 | 需修改 |
| `cancelled` | 已取消 | 放弃报名 |

### 2.2 状态流转图

```
not_started
    ↓
team_profile_check
    ├── 无历史战队 ──→ team_form_filling
    └── 有历史战队 ──→ team_reuse_select
                          ├── 沿用战队 ──→ member_intake
                          ├── 修改战队 ──→ team_form_filling
                          └── 新建战队 ──→ team_form_filling

team_form_filling
    ↓ 完成
member_intake
    ↓ 逐个处理成员
member_profile_check ──→ 逐个member_reuse_select ──→ 逐个member_form_filling
    ↓ 所有成员完成
roster_confirm
    ├── 确认 ──→ submitted
    └── 返回修改 ──→ member_intake

submitted ──→ pending_review ──→ approved/rejected
```

### 2.3 触发条件与动作

| 当前状态 | 触发条件 | 动作 | 下一状态 |
|----------|----------|------|----------|
| `not_started` | 点击战队赛报名 | 查询teams表 | `team_profile_check` |
| `team_profile_check` | 无战队记录 | 推送战队表单 | `team_form_filling` |
| `team_profile_check` | 有战队记录 | 推送复用选择 | `team_reuse_select` |
| `team_reuse_select` | 选择沿用/修改/新建 | 相应操作 | `team_form_filling` 或 `member_intake` |
| `team_form_filling` | 完成战队信息 | 保存teams数据 | `member_intake` |
| `member_intake` | 开始录入成员 | 逐个检查成员资料 | `member_profile_check` |
| `member_profile_check` | 逐个处理 | 复用选择或新建 | `member_reuse_select/form_filling` |
| `member_form_filling` | 完成单个成员 | 保存到players + team_members | 返回`member_intake` |
| `member_intake` | 所有成员完成 | 推送名单确认 | `roster_confirm` |
| `roster_confirm` | 确认名单 | 写入registrations + registration_members | `submitted` |

### 2.4 数据写入

| 状态 | 写入表 | 写入内容 |
|------|--------|----------|
| `team_form_filling` | teams | 战队名称、logo、宣言、队长player_uid |
| `member_form_filling` | players | 成员个人信息 |
| `member_form_filling` | team_members | 战队成员关系、角色(首发/替补) |
| `roster_confirm` | registrations | 战队报名记录 |
| `roster_confirm` | registration_members | 本次赛事出战名单快照 |

## 3. 个人赛卡组提交入口

### 3.1 状态定义

| 状态码 | 状态名 | 说明 |
|--------|--------|------|
| `not_started` | 未开始 | 未触发提交流程 |
| `eligibility_check` | 资格检查 | 检查是否已报名且通过 |
| `not_eligible` | 无资格 | 未报名或审核未通过 |
| `deadline_check` | 截止检查 | 检查是否在截止时间前 |
| `deadline_passed` | 已截止 | 超过截止时间 |
| `form_filling` | 填写中 | 填写4套卡组 |
| `auto_validating` | 自动校验中 | 系统校验卡组格式 |
| `validation_failed` | 校验失败 | 格式/数量/职业不符合 |
| `submitted` | 已提交 | 等待审核 |
| `pending_review` | 审核中 | 管理员审核卡组 |
| `approved` | 已通过 | 卡组有效，获得参赛资格 |
| `rejected` | 已驳回 | 需修改重提 |
| `superseded` | 已替代 | 被新版本替代(补丁后) |

### 3.2 状态流转图

```
not_started
    ↓
eligibility_check ──→ 未报名 ──→ not_eligible (提示先报名)
    ↓ 已报名
    ├── 未审核通过 ──→ not_eligible
    └── 已通过 ──→ deadline_check
                       ├── 已截止 ──→ deadline_passed
                       └── 未截止 ──→ form_filling
                                        ↓
                              auto_validating
                                   ↓
                    ┌──────────────┼──────────────┐
                    ↓              ↓              ↓
            validation_failed  submitted      (保存草稿)
                    ↓              ↓
              返回修改      pending_review
                                  ↓
                          ┌───────┴───────┐
                          ↓               ↓
                      approved        rejected
                          │               ↓
                          │         返回修改
                    (补丁后新建)
                          ↓
                    superseded
```

### 3.3 触发条件与动作

| 当前状态 | 触发条件 | 动作 | 下一状态 |
|----------|----------|------|----------|
| `not_started` | 点击卡组提交 | 查询registrations表 | `eligibility_check` |
| `eligibility_check` | 无报名记录 | 提示先报名 | `not_eligible` |
| `eligibility_check` | 报名未审核通过 | 提示等待审核 | `not_eligible` |
| `eligibility_check` | 已通过 | 检查截止时间 | `deadline_check` |
| `deadline_check` | 已截止 | 提示截止 | `deadline_passed` |
| `deadline_check` | 未截止 | 推送卡组表单 | `form_filling` |
| `form_filling` | 提交 | 自动校验 | `auto_validating` |
| `auto_validating` | 校验失败 | 返回错误详情 | `validation_failed` |
| `auto_validating` | 校验通过 | 写入数据 | `submitted` |
| `submitted` | 系统流转 | 通知管理员 | `pending_review` |
| `pending_review` | 管理员通过 | 更新状态 | `approved` |
| `pending_review` | 管理员驳回 | 说明原因 | `rejected` |
| `rejected` | 修改重提 | 新建deck_submissions V2 | `submitted` |
| `approved` | 补丁后新建版本 | 旧版本标记superseded | `superseded` |

### 3.4 自动校验规则

| 校验项 | 规则 | 失败处理 |
|--------|------|----------|
| 卡组数量 | 必须恰好4套 | 返回错误：需提交4套卡组 |
| 职业重复 | 4套必须不同职业 | 返回错误：职业重复 |
| 代码格式 | 符合炉石卡组代码格式 | 返回错误：代码格式不正确 |
| 图片上传 | 至少上传截图或填写代码 | 返回警告：建议上传截图 |
| 卡组名称 | 非空，建议规范命名 | 返回警告：建议规范命名 |

### 3.5 数据写入

| 状态 | 写入表 | 写入内容 |
|------|--------|----------|
| `auto_validating` → `submitted` | deck_submissions | 提交ID、player_uid、版本号、提交时间 |
| `auto_validating` → `submitted` | decks | 4条记录：职业、卡组名称、代码、图片 |
| `approved` | deck_submissions | 更新状态、审核时间、生效标记 |
| `superseded` | deck_submissions | 旧版本标记为superseded |

## 4. 战队赛卡组提交入口

### 4.1 状态定义

与个人赛类似，主要区别：
- 提交主体是战队而非个人
- 需要11/12套卡组（11职业 + 可能的双职业）
- 由队长统一提交

### 4.2 状态流转

```
not_started
    ↓
eligibility_check ──→ 检查战队报名状态
    ↓
deadline_check
    ↓
form_filling (11/12套卡组)
    ↓
auto_validating
    ├── 校验：数量11/12套
    ├── 校验：职业覆盖要求
    └── 校验：代码格式
    ↓
submitted ──→ pending_review ──→ approved/rejected
```

### 4.3 特殊校验规则

| 校验项 | 规则 | 说明 |
|--------|------|------|
| 卡组数量 | 11或12套 | 标准11职业，可能允许双职业 |
| 职业覆盖 | 必须覆盖所有可用职业 | 确保BP可行性 |
| 提交权限 | 仅限队长提交 | 校验team_members角色 |

### 4.4 数据写入

| 状态 | 写入表 | 写入内容 |
|------|--------|----------|
| `submitted` | deck_submissions | team_uid、版本号、提交时间 |
| `submitted` | decks | 11/12条记录，含职业、名称、代码、图片 |

## 5. 争议提交入口

### 5.1 状态定义

| 状态码 | 状态名 | 说明 |
|--------|--------|------|
| `not_started` | 未开始 | 未触发争议流程 |
| `match_lookup` | 对局查询 | 查询相关对局信息 |
| `match_not_found` | 对局未找到 | 无法定位争议对局 |
| `form_filling` | 填写中 | 填写争议详情 |
| `evidence_upload` | 证据上传 | 上传截图/录像 |
| `submitted` | 已提交 | 争议已提交 |
| `ai_review` | AI初审 | 机器人分析规则与上下文 |
| `ai_recommendation` | AI建议 | 生成处理建议 |
| `admin_review` | 管理员复审 | 等待管理员最终裁决 |
| `resolved` | 已解决 | 争议处理完成 |
| `rejected` | 已驳回 | 申诉不成立 |
| `appealed` | 已上诉 | 对裁决不满继续申诉 |

### 5.2 状态流转图

```
not_started
    ↓
match_lookup ──→ 未找到 ──→ match_not_found (允许手动填写)
    ↓ 找到
form_filling
    ↓
evidence_upload
    ↓
submitted
    ↓
ai_review ──→ ai_recommendation
    ↓
admin_review
    ↓
    ├── 采纳AI建议 ──→ resolved/rejected
    └── 修改裁决 ──→ resolved/rejected

resolved/rejected
    ↓ 不满
appealed ──→ 进入新一轮review
```

### 5.3 触发条件与动作

| 当前状态 | 触发条件 | 动作 | 下一状态 |
|----------|----------|------|----------|
| `not_started` | 私聊"提交争议" | 查询近期对局 | `match_lookup` |
| `match_lookup` | 找到对局 | 预填对局信息 | `form_filling` |
| `match_lookup` | 未找到 | 允许手动输入 | `form_filling` |
| `form_filling` | 提交争议 | 进入证据上传 | `evidence_upload` |
| `evidence_upload` | 完成上传 | 写入数据 | `submitted` |
| `submitted` | 系统触发 | AI分析规则 | `ai_review` |
| `ai_review` | 完成分析 | 生成建议 | `ai_recommendation` |
| `ai_recommendation` | 推送管理员 | 等待裁决 | `admin_review` |
| `admin_review` | 管理员裁决 | 更新状态 | `resolved`/`rejected` |
| `resolved`/`rejected` | 选手上诉 | 重新审核 | `appealed` → `admin_review` |

### 5.4 数据写入

| 状态 | 写入表 | 写入内容 |
|------|--------|----------|
| `submitted` | disputes | 争议ID、对局ID、争议类型、详细说明、诉求、证据列表 |
| `ai_recommendation` | disputes | AI分析结果、建议处理方案 |
| `resolved`/`rejected` | disputes | 最终裁决、裁决理由、处理时间 |
